# Роборука

Проект представляет сервер для проверки статуса кнопки роборуки-держателя. Статус можно посмотреть и изменить через адресную строку браузера.

![Эскиз руки-держателя](Эскиз.jpg)

## Описание проекта

Рука-держатель нужна для удержания предмета в ладони. На ладони расположена кнопка. Когда предмет попадает в руку, он нажимает кнопку, после чего четыре пальца сгибаются и охватывают предмет. Большой палец остается неподвижным и служит опорой.

Сжатие пальцев происходит за счет нити, которая наматывается на катушку. Катушку вращает сервопривод. При намотке нить тянет пальцы в положение сжатия. При разматывании нити пружины в шарнирах возвращают пальцы в исходное положение.

На запястье находится LED-индикатор:

- если кнопка не нажата, индикатор не горит;
- если кнопка нажата, но предмета нет, индикатор красный;
- если кнопка нажата и предмет есть, индикатор зеленый.

## Что делает сервер

Сервер хранит состояние кнопки и предмета. Через URL-команды можно изменить это состояние. После изменения сервер пересчитывает состояние роборуки:

- нажата ли кнопка;
- есть ли предмет в руке;
- сжаты ли пальцы;
- какой цвет у LED-индикатора;
- итоговое состояние.

## Как запустить

Сначала нужно запустить сервер. Без этого ссылки `http://127.0.0.1:8000/...` работать не будут.

1. Скачать файлы проекта.
2. Открыть терминал в папке проекта.
3. Выполнить команду:

```bash
python app.py
```

4. Не закрывать терминал.
5. Открыть в браузере:

```text
http://127.0.0.1:8000/
```

## Команды через адресную строку

После запуска сервера можно вводить команды в адресную строку браузера.

| Команда | Что делает |
| --- | --- |
| `http://127.0.0.1:8000/` | главная страница |
| `http://127.0.0.1:8000/status` | показать текущий статус текстом |
| `http://127.0.0.1:8000/status_on` | включить статус кнопки |
| `http://127.0.0.1:8000/status_off` | выключить статус кнопки |
| `http://127.0.0.1:8000/object_on` | указать, что предмет находится в руке |
| `http://127.0.0.1:8000/object_off` | указать, что предмета в руке нет |
| `http://127.0.0.1:8000/reset` | сбросить состояние |

Дополнительно оставлены команды `/button_on`, `/button_off`, `/knopka_on`, `/knopka_off`. Они делают то же самое, что `/status_on` и `/status_off`.

## Логика работы

| Кнопка | Предмет в руке | Пальцы | LED | Итог |
| --- | --- | --- | --- | --- |
| нет | нет | разжаты | не горит | рука разжата |
| да | нет | сжаты | красный | ложное нажатие |
| да | да | сжаты | зеленый | предмет удерживается |

Если выполнить команду `/object_on`, сервер автоматически считает кнопку нажатой, потому что предмет давит на кнопку в ладони.

## Проверка

После запуска сервера можно открыть:

```text
http://127.0.0.1:8000/status_on
```

Потом открыть:

```text
http://127.0.0.1:8000/status
```

В статусе должно появиться:

```text
Кнопка нажата: да
Предмет в руке: нет
Пальцы сжаты: да
LED-индикатор: красный
Итог: ложное нажатие
```
