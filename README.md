# Рука-держатель

Проект представляет веб-интерфейс для просмотра состояния роборуки. Состояние кнопки и предмета хранится и обрабатывается на сервере Python.

![Эскиз руки-держателя](Эскиз.jpg)

## 1. Назначение проекта

Рука-держатель предназначена для удержания предмета в ладони. Пример предмета: стакан, банка или небольшой цилиндрический объект.

На ладони находится кнопка. Если предмет попадает в руку, он нажимает кнопку. После нажатия четыре пальца сгибаются и охватывают предмет. Большой палец не двигается и служит неподвижной опорой.

На запястье находится LED-индикатор:

- кнопка не нажата - LED не горит;
- кнопка нажата, но предмета нет - LED горит красным;
- кнопка нажата и предмет есть - LED горит зеленым.

## 2. Механическая часть роборуки

В конструкции есть четыре подвижных пальца и один статичный большой палец. Подвижные пальцы имеют шарниры с возвратными пружинами.

Сжатие происходит за счет нити:

1. сервопривод вращает катушку;
2. катушка наматывает нить;
3. нить тянет четыре пальца;
4. пальцы сгибаются и охватывают предмет.

Разжатие происходит обратно:

1. катушка разматывает нить;
2. натяжение уменьшается;
3. пружины в шарнирах возвращают пальцы в исходное положение.

Исходный эскиз конструкции находится в файле `Эскиз.jpg`.

## 3. Что делает программа

Программа запускает локальный сервер на Python. Через браузер можно изменить состояние кнопки и предмета. Сервер сохраняет это состояние в переменной `HAND_STATE`, рассчитывает состояние пальцев и LED-индикатора, а затем возвращает страницу с результатом.

На странице выводится 5 строк:

1. кнопка нажата: да / нет;
2. предмет в руке: да / нет;
3. пальцы сжаты: да / нет;
4. LED-индикатор: не горит / красный / зеленый;
5. итоговое состояние руки.

Если включить состояние `Предмет: да`, сервер автоматически считает, что кнопка тоже нажата, потому что предмет давит на кнопку в ладони.

## 4. Архитектура работы

```text
Браузер / микроконтроллер
        |
        | HTTP GET
        v
Python-сервер app.py
        |
        | хранит состояние HAND_STATE
        v
Расчет пальцев и LED
        |
        v
HTML-страница или текстовый статус
```

## 5. Логика состояний

| Кнопка | Предмет в руке | Пальцы | LED | Итог |
| --- | --- | --- | --- | --- |
| нет | нет | разжаты | не горит | рука разжата |
| да | нет | сжаты | красный | ложное нажатие |
| да | да | сжаты | зеленый | предмет удерживается |

## 6. Доступные команды

Команды вводятся в адресной строке браузера после запуска сервера.

| URL | Что делает |
| --- | --- |
| `/` | главная страница с текущим состоянием |
| `/status` | показывает текущее состояние текстом |
| `/button_on` | устанавливает состояние кнопки `да` |
| `/button_off` | устанавливает состояние кнопки `нет` и убирает предмет |
| `/knopka_on` | то же самое, что `/button_on` |
| `/knopka_off` | то же самое, что `/button_off` |
| `/object_on` | устанавливает состояние предмета `да`; кнопка тоже становится `да` |
| `/object_off` | устанавливает состояние предмета `нет` |
| `/reset` | сбрасывает состояние: кнопка `нет`, предмет `нет` |

## 7. Примеры использования

Открыть главную страницу:

```text
http://127.0.0.1:8000/
```

Включить кнопку:

```text
http://127.0.0.1:8000/button_on
```

Показать, что предмет находится в руке:

```text
http://127.0.0.1:8000/object_on
```

Получить текстовый статус:

```text
http://127.0.0.1:8000/status
```

Сбросить состояние:

```text
http://127.0.0.1:8000/reset
```

## 9. Как запустить

Нужен Python 3. Дополнительные библиотеки устанавливать не нужно.

1. Скачать папку проекта.
2. Открыть терминал в папке проекта.
3. Запустить сервер:

```bash
python app.py
```

4. Открыть сайт в браузере:

```text
http://127.0.0.1:8000
```

Ссылка открывается только при запущенном сервере. Окно терминала с командой `python app.py` нужно оставить открытым.

## 10. Почему это серверная реализация

Кнопки на странице являются обычными ссылками на серверные URL. Когда пользователь нажимает ссылку, браузер отправляет запрос на Python-сервер. Сервер меняет состояние, пересчитывает результат и заново отдает страницу.

Это значит, что информация о кнопке и предмете находится на стороне сервера, а не только в браузере пользователя.
